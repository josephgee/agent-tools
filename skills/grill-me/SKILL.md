# Grill Me

Interview the user relentlessly about every aspect of the plan, design, or idea until you reach a shared understanding. Ask one question at a time — don't front-load a checklist of everything you're going to ask.

## How to grill

- **Walk the decision tree branch by branch.** Resolve every dependency within a branch before moving to the next one — don't jump around between unrelated threads.
- **Explore before asking.** If a question can be answered by reading the codebase, docs, or other available material instead of asking the user, do that instead. Only ask what you genuinely can't discover yourself.
- **Recommend an answer with every question.** State your own recommended answer alongside each question so the user can just say "yes" instead of reasoning through it from scratch — but make it easy for them to disagree and go a different way.
- **Push past the first plausible answer.** Keep probing edge cases, failure modes, and unstated assumptions. A satisfying-sounding first answer often has a gap underneath it — that's what the grilling is for.
- **The method is domain-agnostic.** This works the same for coding plans (architecture, data model, error handling, edge cases) as for non-coding ideas (a course outline, a business plan, an essay's argument) — adjust what you ask about, not how you ask.

## Wrapping up

Keep going until every branch is resolved — no open questions, no hand-waved decisions. This is often a long conversation (30–45 minutes for a substantial plan); don't rush to the summary before the tree is actually resolved.

Then summarize: the shared understanding reached, the key decisions made and why, and anything explicitly deferred or left open.
